二、关键发现：文件系统就是通信总线2. Key Discovery: The File System IS the Communication Bus

所有 Agent 共享同一个文件系统。我们发现 PM Studio 在磁盘上存储了远比预期多的数据——包括完整对话历史、Session 元数据、Memory 数据库。All agents share the same file system. We discovered PM Studio stores far more data on disk than expected — including complete conversation history, session metadata, and the memory database.

七、实测中发现的 Bug 及修复7. Bug Discovery & Fix During Testing

跨月 Session 扫描盲区 — 一个差点让情报系统失效的设计缺陷。Cross-month session scanning blind spot — a design flaw that nearly broke the intelligence system.

🐛 Bug

用户刚让网球情报官做了任务，Meta-Agent 没有感知到。原因：新消息发生在一个 3 月创建的旧 session 中，但 Meta-Agent 只检查了 202604/ 目录。User just assigned a task to Tennis Scout, but Meta-Agent didn't detect it. Cause: New messages went to an old March session, but Meta-Agent only checked the 202604/ directory.

❌ 修复前❌ Before Fix

// Only checked current month — WRONG!
1. Read 202604/index.json
2. Pick most recent session
3. → Missed session in 202603/ with
   fresh last_updated timestamp

✅ 修复后✅ After Fix

// Scan ALL months + sort by time
1. List ALL month directories
2. Read EVERY index.json
3. Merge all sessions into one list
4. Sort by last_updated DESC
5. → Catches activity in any month

// Shortcut: filesystem LastWriteTime
Get-ChildItem -Recurse "*.json" |
  Sort LastWriteTime -Desc
根因：PM Studio 的 chatSessionManager.ts 按月分片存储但不迁移文件。Session 留在创建月份，收到新消息时只更新 last_updated。Root cause: PM Studio's chatSessionManager.ts shards by month but never migrates files. Sessions stay in their creation month; only last_updated gets refreshed.

十、设计启示与下一步10. Design Insights & Next Steps

🏗️ 文件系统即协议🏗️ File System as Protocol

在没有 native IPC 的架构约束下，文件系统是天然的异步消息总线。JSONL append-only + 定期读取 = 简洁可靠的 pub/sub。Without native IPC, the file system serves as a natural async message bus. JSONL append-only + periodic reading = simple and reliable pub/sub.

📊 渐进式信息获取📊 Progressive Information Retrieval

不要一次性加载所有信息。500 tokens 能解决的问题不要花 50K。这是 context engineering 在 agent 协作中的映射。Don't load everything at once. If 500 tokens solve the problem, don't spend 50K. This is context engineering applied to agent coordination.

🔑 能力即权限🔑 Capability as Permission

通过 Skill 分配控制谁是"下属"。不需要 ACL——Skill 列表本身就是权限矩阵。Skill assignment controls who is a "subordinate." No ACL needed — the skill list IS the permission matrix.

🔄 从实践中学习🔄 Learn from Practice

跨月扫描 bug 说明——设计必须经过实测验证。Meta-Agent 的知识库是活的文档，随实践持续进化。The cross-month bug proves — designs must be validated through testing. The Meta-Agent's KB is a living document that evolves with practice.
